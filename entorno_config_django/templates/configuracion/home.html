🚀 ¿Qué es Django?

Es un framework web de alto nivel escrito en Python que fomenta un desarrollo rápido y un diseño limpio y pragmático.

Django sigue la filosofía "Batteries Included" (Baterías Incluidas), proporcionando de fábrica casi todo lo necesario para aplicaciones empresariales listas para producción: autenticación, ORM, enrutamiento, seguridad y panel de administración.

🎯 ¿Por qué elegirlo?
  • Velocidad de Desarrollo: Diseñado para llevar aplicaciones desde el concepto a la producción en el menor tiempo posible.
  • Seguridad por Defecto: Protege de forma automática contra ataques SQL Injection, XSS, CSRF y Clickjacking.
  • Escalabilidad: Utilizado por las empresas más grandes del mundo (Instagram, Pinterest, Disqus) por su capacidad para manejar cargas masivas de datos.
⚙️ Desarrollo v/s Producción

El entorno de Django es altamente flexible y seguro. Diferencia claramente las configuraciones de desarrollo y producción:

DEBUG = True (Desarrollo: muestra errores detallados)
DEBUG = False (Producción: oculta detalles técnicos)
ALLOWED_HOSTS = ['midominio.com']

🏗️ Arquitectura de Django: El Patrón MTV

Aunque comúnmente se le asocia con el clásico MVC (Modelo-Vista-Controlador), Django implementa una variante denominada MTV (Modelo-Template-Vista). En esta arquitectura, el framework asume el rol del controlador.

📁 El Modelo (M)

Es la capa de acceso a los datos. Define la estructura de tu base de datos mediante clases de Python, abstrayéndote de escribir SQL directamente gracias al potente ORM (Object-Relational Mapping) de Django.

🧠 La Vista (V)

Representa el controlador lógico de la aplicación. Se encarga de procesar la solicitud del usuario (request), interactuar con el Modelo para obtener los datos y retornar una respuesta (normalmente renderizando un Template).

🎨 El Template (T)

Es la capa de presentación. Archivos HTML enriquecidos con el motor de plantillas de Django (DTL), que permite inyectar datos dinámicos provenientes de la vista usando bucles, condiciones y filtros avanzados.

⚡ El Principio DRY (Don't Repeat Yourself)

El principio "No te repitas" está en el corazón de Django. Su objetivo es reducir la duplicación de información y código de todo tipo:

  • Herencia de Componentes: La posibilidad de heredar plantillas HTML (como base.html) evita copiar cabeceras, menús y pies de página.
  • Modelos Únicos: Defines tus datos una sola vez en models.py y Django crea las tablas, formularios de admin y consultas automáticamente.
🛣️ El Enrutador de Django (URLs)

El enrutador actúa como el mapa de tu sitio. Mapea expresiones o patrones de URL directamente a funciones de vista de Python.

Cuando llega una petición web, Django analiza el archivo urls.py principal del proyecto, delega a las aplicaciones y ejecuta la vista correspondiente en milisegundos.

🛡️ ¿Por qué usar Entornos Virtuales?

Un entorno virtual es un directorio auto-contenido que contiene una instalación específica de Python y un conjunto de librerías independientes de las del resto del sistema.

Ventajas Clave:

  • Aislamiento de Entorno: Evita conflictos entre dependencias. Si el Proyecto A requiere Django 4.2 y el Proyecto B Django 6.0, los entornos virtuales resuelven esto fácilmente.
  • Estructura Minimalista y Flexible: Cada entorno virtual pesa pocos megabytes originalmente y solo incluye las librerías estrictamente necesarias para el proyecto.
  • Manejo de Diferentes Versiones: Puedes tener entornos virtuales que utilicen distintas versiones de Python (3.9, 3.10, 3.11, etc.).
  • Velocidad de Desarrollo: Al instalar dependencias localmente, los comandos de Django y las pruebas corren en un contexto aislado, limpio de interferencias del sistema global.

El Peligro de las Instalaciones Globales:

Instalar librerías directamente en el entorno global de tu sistema (usando pip install directamente) ensucia el sistema operativo, requiere privilegios de administrador y puede romper otras herramientas o scripts del sistema al sobreescribir dependencias globales.

💻 Comandos Reales en la Consola

Sigue esta guía paso a paso para inicializar y controlar un entorno virtual en tu propio computador.

📦 1. Creación con el comando venv

Para crear un entorno virtual, abres una consola en la raíz de tu proyecto y ejecutas:

python -m venv venv

Esto creará una carpeta llamada venv que contiene copias limpias de los ejecutables de Python y las herramientas básicas de empaquetado.

⚡ 2. Activación

Para indicarle a tu consola que debe usar este entorno aislado:

En Windows (PowerShell):

.\venv\Scripts\Activate.ps1

En macOS/Linux (Bash/zsh):

source venv/bin/activate
🚪 3. Salir (Desactivar)

Cuando termines de trabajar en tu proyecto, puedes salir del entorno virtual escribiendo:

deactivate

Esto restaura inmediatamente las variables de entorno de tu sistema, volviendo al Python global.
